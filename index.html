Hermeer

Setup guides for Desktop and CLI modes

These guides walk you through connecting Hermeer to a depth companion on your computer. If you've never used Terminal before, that's completely fine — every step is explained.

Desktop ~15 minutes CLI ~30 minutes

Desktop Mode

Desktop mode connects Claude Desktop (Anthropic's computer app) to your Hermeer data. Claude reads your entries, pulls, and threads — then writes interpretations back. They appear in the Hermeer app automatically.

What you'll need

  • A Mac or Windows computer
  • A Claude Pro or Max subscription from Anthropic
  • About 15 minutes
Step 1

Open Terminal

Terminal is a built-in app on your Mac where you can type commands. You'll use it for a few of these steps.

How to find it: Press Command + Space (the Spotlight search bar appears). Type Terminal and press Enter.

A window with a dark or white background and a blinking cursor will open. That's Terminal. You'll paste commands into this window.

On Windows: use PowerShell instead of Terminal. Press the Windows key, type "PowerShell", and open it.

Step 2

Install Node.js

Node.js is a free tool your computer needs to run the Hermeer connector. You only install it once.

Mac: Paste this into Terminal and press Enter:

brew install node

If you see "command not found: brew" — that's okay. It means you need to install Homebrew first. Paste this into Terminal and press Enter, then try the command above again:

/bin/bash -c "$(curl -fsSL https://raw.githubusercontent.com/Homebrew/install/HEAD/install.sh)"

Alternatively, you can download Node.js directly from nodejs.org — click the big green button and install it like any other app.

Windows: Go to nodejs.org, download the installer, and run it. Use all the default options.

Step 3

Install the Hermeer connector

This downloads a small piece of software that lets Claude Desktop talk to your Hermeer data.

Paste this into Terminal and press Enter:

git clone https://github.com/hermeerapp/hermeer-mcp-server.git ~/.hermeer-mcp-server && cd ~/.hermeer-mcp-server && npm install && npm run build

You'll see a bunch of text scroll by. When it finishes and you see the blinking cursor again, it worked.

If you see errors mentioning "git: command not found" — install the Xcode command line tools first. Paste this and press Enter: xcode-select --install. A popup will appear — click Install, wait for it to finish, then try the command above again.

Step 4

Download Claude Desktop

If you don't have it yet, download Claude Desktop from Anthropic.

Mac: Go to claude.ai/download, download the Mac version, and drag it into your Applications folder.

Windows: Same link — download the Windows version and run the installer.

Open Claude Desktop and sign in with your Anthropic account.

Step 5

Connect the Hermeer connector to Claude Desktop

Claude Desktop needs to know about the connector you just installed. You'll do this by editing a settings file.

Mac: Paste this into Terminal and press Enter:

mkdir -p ~/Library/Application\ Support/Claude && cat > ~/Library/Application\ Support/Claude/claude_desktop_config.json << 'EOF' { "mcpServers": { "hermeer": { "command": "node", "args": ["$HOME/.hermeer-mcp-server/dist/index.js"] } } } EOF

If you already have other MCP servers configured in Claude Desktop, you'll need to merge this into your existing config file rather than replacing it. Open the file at ~/Library/Application Support/Claude/claude_desktop_config.json and add the "hermeer" entry inside "mcpServers".

Now quit Claude Desktop completely (Cmd+Q on Mac) and reopen it.

Step 6

Set up your Depth Work project

In Claude Desktop:

  1. Click Projects in the sidebar
  2. Click Create Project
  3. Name it Depth Work
  4. Click the project, then click Set project instructions
  5. Paste the companion instructions from Hermeer. To get them: open Hermeer on your phone, go to Settings → Depth Companion → Copy Project Instructions

This is the one time you'll need to move text from your phone to your computer. The easiest ways: email it to yourself, paste it into a note (Apple Notes syncs between devices), or use AirDrop.

Step 7

Try it

Open a new chat inside your Depth Work project and say:

"Read my current state and tell me what's alive."

Claude will read your practice data through the connector and respond with a depth reading. Any interpretations it writes will appear in the Hermeer app on the entries they reference.

That's it. Desktop mode is set up. Claude Desktop can now read your entries, pulls, dreams, and threads — and write interpretations back to the app.

CLI Mode

CLI mode adds automation on top of Desktop mode — morning briefings, evening readings, and weekly syntheses that run on a schedule and appear in the app. It uses Claude Code, Anthropic's command-line tool.

Complete Desktop mode first. CLI mode builds on top of it.

What you'll need

  • Desktop mode already set up (see above)
  • A Claude Max subscription (CLI mode uses Claude Code, which requires Max)
  • About 15 more minutes
Step A

Install Claude Code

Claude Code is a command-line version of Claude that can run automated tasks. Paste this into Terminal:

npm install -g @anthropic-ai/claude-code

When it finishes, type claude and press Enter to make sure it works. You should see Claude Code start up. Type /exit to close it.

Step B

Run the setup script

This script sets up your depth practice commands. It will ask you a few questions (birth data, sync folder location) and create everything automatically.

Paste this into Terminal:

curl -sO https://raw.githubusercontent.com/hermeerapp/hermeer-mcp-server/main/scripts/setup-depth-practice.sh && bash setup-depth-practice.sh

Follow the prompts. When it finishes, close Terminal and reopen it (so the new commands are available).

Step C

Try your new commands

You now have three commands you can type in Terminal anytime:

  • morning — a morning briefing based on your practice, transits, and recent themes
  • evening — an evening reflection and day-arc reading
  • sunday — a weekly reading and synthesis (runs automatically on Sunday mornings, or anytime you want)

Try typing morning in Terminal and pressing Enter. Claude will read your data and write a briefing that appears in the Hermeer app.

That's it. CLI mode is set up. Your morning and evening commands are ready, and weekly readings will run automatically on Sundays.

Troubleshooting

"command not found" errors

This usually means a tool wasn't installed properly. Try closing Terminal completely (Cmd+Q) and reopening it. If the problem persists, go back to the step where you installed that tool and try again.

Interpretations aren't appearing in the app

Make sure your Hermeer sync folder is set up. In the Hermeer app: Settings → Sync. The connector looks for your data in the sync folder — if sync isn't configured, there's nothing to read.

Claude Desktop doesn't see the connector

After editing the config file (Step 5), you must completely quit Claude Desktop (Cmd+Q, not just close the window) and reopen it. Check that the config file path is correct by pasting this in Terminal:

cat ~/Library/Application\ Support/Claude/claude_desktop_config.json

You should see the "hermeer" entry. If the file is empty or missing, go back to Step 5.
